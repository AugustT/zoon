An Introduction to the zoon package

Introduction

Zoon is a package to aid reproducibility and between model comparisons in species distribution modelling. The aim is for each step in an analysis to be a 'module'. These modules will include:

  • Data collection of occurrence and environmental covariate data from online databases.
  • Preprocessing steps such as removal of spatial autocorrelation in the data or generation of background pseudoabsences.
  • The fitting of models.
  • Model diagnostics.
  • Model output and vizualisation.

We aim for the creation of these modules to be as simple as possible.

Basic usage

library(zoon)

# define the extent in lat and long
uk.extent <- c(xmin = -10,
              xmax = 10,
              ymin = 45,
              ymax = 65)

Then run two workflows. Both workflows use the module 'AnophelesPlumbeus' to get \emph{Anopheles plumbeus} records from GBIF. 'AirNCEP' then collects air temperature records from NCEP. The module 'OneHundredBackground' uses 100 random bakground points as our pseudoabsense data.

The two workflows run different SDMs. The first runs a logistic regression while the second runs a random forest. Finally, we use the module 'SomeTimePlaceMap' to predict occurence across the extent we already defined and for the present time.

# run a workflow, using the logistic regression model
ans1 <- workflow(extent = uk.extent,
                 occurrence.module = 'AnophelesPlumbeus',
                 covariate.module = 'AirNCEP',
                 process.module = 'OneHundredBackground',
                 model.module = 'LogisticRegression',
                 map.module = 'SameTimePlaceMap')

Some points to note about reproducibility. The whole output object should contain all the information needed to reproduce the analysis. So for example, the data is saved. Therefore, even if GBIF gets new data, the exact data used in this analysis is stored.
